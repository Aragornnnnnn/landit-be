# LAN-178 작업 계획

## 범위

- 세션 최종 상세 피드백 저장이 완료되면 연결된 원본 사용자 메시지의 피드백 처리 상태를 `PREPARING`에서 `COMPLETED`로 갱신한다.
- 기존 `FAILED` 상태와 피드백 문구·점수·판정 정책, 활동 요약 집계는 변경하지 않는다.
- 배포 전에 이미 저장된 상태 불일치 데이터의 일괄 보정은 이번 코드 hotfix에 포함하지 않는다.

## 구현 및 검증

- [x] 상세 피드백과 원본 메시지의 완료 상태를 함께 검증하는 통합 테스트를 추가한다.
- [x] 최종 피드백 저장 트랜잭션에서 원본 사용자 메시지 상태를 조건부로 완료 처리한다.
- [x] 기존 실패 처리와 세션 피드백·시나리오 진행도 저장 회귀 테스트를 실행한다.
- [x] 사용자 메시지가 여러 건인 세션도 모든 상세 피드백과 원본 메시지가 완료 상태인지 검증한다.
- [x] `./gradlew check`를 실행한다.

검증 결과. `./gradlew check`가 통과했다.
